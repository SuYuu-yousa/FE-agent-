# 浏览器进程

浏览器通常至少包含浏览器主进程、渲染进程、网络相关进程或服务、GPU 进程，另外还可能有插件进程、扩展进程、音视频相关进程、存储和 utility 进程

主进程就是管多少个标签页之类的

渲染进程一般每个页面一个，画这个页面

网络进程字面意思

# 页面怎么来？

你现在输入了一个url

### 1.从url输入 到 准备请求

浏览器拿到 URL 后，先判断输入内容是不是一个可导航的地址(不是可能会转给搜索引擎)，再把它解析成规范化的 URL 结构，然后判断这次变化是不是一次真正的页面导航，比如只是 hash 变化就未必会重新请求；如果需要导航，它会优先检查本地状态和可复用资源，比如缓存、已有连接、已有网络信息，确认之后才进入真正的网络获取流程，也就是后面的 DNS、建连、发请求。你可以先把最开始记成一句话：识别输入、规范 URL、判断导航、尝试复用、准备请求。

### 2.发请求到返回入口文件











流程很简单：

1. 浏览器发请求
2. 服务器返回 HTML
3. 返回内容里分两部分理解：
   - **响应头**：先告诉浏览器“这页面该怎么处理”
   - **响应体**：真正的 HTML 内容

比如服务器会先说：

- 这个页面是什么编码
- 能不能执行外部脚本
- 能不能加载第三方资源
- 要不要缓存

然后浏览器再去解析 HTML。
