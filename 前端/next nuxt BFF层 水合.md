# 基本概念-CSR SSR SSG

- **Next.js** 是基于 **React** 的框架

- **Nuxt.js** 是基于 **Vue** 的框架

- 它们常被叫做 **全栈框架**

- 它们支持 **SSR / SSG / CSR** 等页面渲染方式

- **确实会让 React/Vue 的一部分代码在服务器运行**

- 但不是简单地“把整个前端搬到服务器”，而是：

  - 有些代码在服务器跑

  - 有些代码在浏览器跑

  - 两边协作完成网页


------

## 什么叫“全栈框架”？

> **全栈框架 = 不只管前端页面，还顺带帮你处理服务端能力的框架**

普通 React / Vue 更多像是“写界面”的工具。
而 Next / Nuxt 不只是写界面，还会帮你处理：

- 路由 页面渲染  服务端渲染
- 静态生成  接口/API  部署优化
- SEO 相关能力

所以它不是单纯前端框架，而是“前端 + 一部分后端能力”都给你打包好了。

------

| 方式    | 一句话总结                       |
| ------- | -------------------------------- |
| **SSR** | 页面是“访问时在服务器现做的”     |
| **SSG** | 页面是“打包时提前做好的”         |
| **CSR** | 页面是“浏览器自己用 JS 画出来的” |

------

#  Next / Nuxt 是什么？

你可以理解成：

> 它们给 React/Vue 提供了“多种渲染方式 + 服务端能力 + 工程化能力”

它们并不是只做 SSR，实际上它们帮你处理了：

- **页面路由**
- **服务端渲染 SSR**
- **静态生成 SSG**
- **客户端渲染 CSR**
- **接口/API**
- **打包构建**
- **性能优化**
- **SEO 支持**

所以你开发网站时，不用从零拼各种工具。

------

# “让 React 和 Vue 在服务器上运行”这句话对吗？

**对，但不完整。**

更准确说法是：

> Next / Nuxt 可以让 React / Vue 组件在服务器端先执行一次，生成 HTML，再把结果发给浏览器。

注意重点不是“永远在服务器跑”，而是：

- **首轮渲染** 可以在服务器
- **后续交互** 主要还是在浏览器

------

## 举个流程例子

你访问一个商品页：

### 使用 Next SSR 时

1. 浏览器请求 `/product/123`
2. Next 服务器拿到请求
3. 去数据库/接口取商品数据
4. 用 React 组件生成 HTML
5. 把 HTML 发给浏览器
6. 浏览器先显示商品信息
7. JS 加载后，按钮点击、加入购物车等交互开始生效

所以是：

- **HTML 先由服务器生成**
- **交互后由浏览器接管**

------

# 同构 / 通用”思想？

你以后可能会听到两个词：

- **同构**
- **isomorphic**
- **universal rendering**

大意是：

> 同一套组件代码，既能在服务器运行，也能在浏览器运行

比如同一个 React 组件：

```
export default function Page() {
  return <h1>Hello</h1>
}
```

这段组件代码：

- 可以在服务器执行，生成 `<h1>Hello</h1>`
- 也可以在浏览器执行，绑定事件、完成交互

这就是 Next/Nuxt 很核心的思想。

------

# 什么时候用 SSR？什么时候用 SSG？什么时候用 CSR？

| 渲染方式                          | 适合场景                         | 典型页面                                         | 主要原因                                                     |
| --------------------------------- | -------------------------------- | ------------------------------------------------ | ------------------------------------------------------------ |
| **SSR**（Server-Side Rendering）  | 内容经常变化，且需要 SEO         | 电商商品详情、新闻网站、搜索结果页、个性化首页   | 首屏加载快；搜索引擎能直接看到内容；数据更新频繁，适合按请求实时生成 |
| **SSG**（Static Site Generation） | 内容更新不频繁，可提前生成       | 博客文章、官方文档、公司官网、落地页             | 页面可以在构建时生成；访问速度快；服务器压力小；不需要每次请求都实时渲染 |
| **CSR**（Client-Side Rendering）  | 更偏应用型，SEO 不重要，交互复杂 | 后台管理系统、数据看板、内部系统、登录后用户中心 | 页面交互复杂；更适合在浏览器端处理状态和逻辑；通常不依赖搜索引擎收录 |
